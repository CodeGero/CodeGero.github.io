Reproducible Python Builds: Why Hash-Pinned Requirements Beat Version Ranges

July 11, 2026 · Kryptorious Tools

A requirements.txt that says requests>=2.0 does not describe a build. It describes a range of possible builds that depend on the clock, the mirror, and whatever the maintainer shipped that morning. For local scripts that is fine. For anything you ship, deploy, or hand to a team, it is a silent liability.

The failure mode

Version ranges resolve at install time. Two engineers running the same requirements.txt three days apart can end up with different transitive dependencies if a sub-dependency released a new minor version in between. The build is no longer reproducible: same inputs (requirements.txt), different outputs (installed tree). When a bug appears, "works on my machine" becomes literally true and completely useless.

This is not theoretical. Supply-chain incidents where a compromised release propagates through permissive ranges are a documented, recurring class of problem. The mitigation is not exotic: pin exact versions and their hashes.

The fix: hash-pinned requirements

pip supports hash-checking mode. A line like:

requests==2.31.0 --hash=sha256:abcdef...

means pip refuses to install that package unless the downloaded artifact's hash matches. A compromised or substituted release fails closed instead of silently shipping. This is the same guarantee class behind SLSA provenance expectations, applied at the Python layer.

Doing it without the busywork

Manually computing hashes for a deep dependency tree is tedious, which is exactly why it gets skipped. Two tools remove the friction:

  • pip-tools (pip-compile) — locks transitive deps to exact versions with hashes.
  • pip-audit — scans an installed or locked environment against the PyPI vulnerability database. (Verified live: pypi.org/pypi/pip-audit/json returns HTTP 200 — the package is published and maintained.)

Wire both into CI and a green build means: exact versions, verified hashes, zero known-vulnerable pins. That is a reproducible build you can defend.
